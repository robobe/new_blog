    Cheat sheet

    Clone

    git clone copies a remote repository to your machine and creates a local working directory.

    Clone the default branch:

    1
    git clone https://github.com/user/repo.git

    Clone a specific branch:

    1
    git clone --branch feature/login https://github.com/user/repo.git

    Clone into a target folder:

    1
    git clone https://github.com/user/repo.git my-target-folder

    Clone a specific branch into a target folder:

    1
    git clone --branch feature/login https://github.com/user/repo.git my-target-folder

    Clone a specific tag:

    1
    git clone --branch v1.0.0 https://github.com/user/repo.git

    Clone the repository and all submodules in one line:

    1
    git clone --recurse-submodules https://github.com/user/repo.git

    Push and pull

    git push uploads commits from your local branch to a remote repository. It does not upload uncommitted file changes.

    Push commits from the current branch:

    push current branch
    1
    git push

    Push a new local branch and set the remote tracking branch:

    push new branch
    1
    git push -u origin feature/login

    Push a local branch to a different remote branch name:

    push to different remote branch
    1
    git push origin local-branch:remote-branch

    Push tags:

    push tags
    1
    git push --tags

    Delete a remote branch:

    delete remote branch
    1
    git push origin --delete feature/login

    Warning

    Avoid git push --force on shared branches. If you must update a rewritten branch, prefer git push --force-with-lease because it refuses to overwrite remote work you have not fetched.

    Force push after rewriting local history:

    safer force push
    1
    git push --force-with-lease

    git pull downloads changes from a remote branch and updates your current branch. By default, pull is similar to git fetch followed by git merge.

    Pull the latest changes:

    pull latest changes
    1
    git pull

    Pull from a specific remote branch:

    pull specific branch
    1
    git pull origin main

    Note

    A normal git pull may create a merge commit when your local branch and the remote branch both have new commits. If Git can fast-forward your branch, no merge commit is created.

    Pull without creating a merge commit:

    pull only if fast-forward is possible
    1
    git pull --ff-only

    Pull with rebase instead of merge:

    pull with rebase
    1
    git pull --rebase

    If you have uncommitted changes when pulling, Git tries to keep them. If the remote changes touch the same files, Git may stop the pull and ask you to commit, stash, or discard your local changes first.

    Check your local changes before pulling:

    check working tree
    1
    git status

    Stash uncommitted changes before pulling:

    stash before pull
    1
2
3
    git stash push -m "work before pull"
git pull
git stash pop

    Commit local changes before pulling:

    commit before pull
    1
2
3
    git add .
git commit -m "save local changes"
git pull

    Pull with automatic stash:

    pull with auto stash
    1
    git pull --rebase --autostash

    Note

    --autostash temporarily stashes your uncommitted changes, pulls the remote changes, then reapplies your work. Conflicts can still happen when the same lines changed locally and remotely.

    git pull updates the branch you are currently on. To review remote changes in a new branch, create the review branch first.

    Create a review branch from the remote branch:

    create review branch from remote
    1
2
    git fetch origin
git switch -c review/main origin/main

    Pull into an existing review branch:

    pull into review branch
    1
2
    git switch review/main
git pull origin main

    After reviewing, commit your own changes on the review branch:

    commit after review
    1
2
    git add .
git commit -m "review remote changes"

    Fetch remote changes without merging into your current branch:

    fetch without changing current branch
    1
    git fetch origin

    Show local commits that are not pushed yet:

    show unpushed commits
    1
    git log origin/main..HEAD --oneline

    Show remote commits that you have not pulled yet:

    show unpulled commits
    1
    git log HEAD..origin/main --oneline

    branch

    Git branches let you work on features, fixes, or experiments without changing the main code line.

    Create a new branch:

    create branch
    1
    git branch feature/login

    Create a new branch and switch to it:

    create and switch branch
    1
    git switch -c feature/login

    Switch between branches:

    switch branch
    1
    git switch main

    Switch to a remote branch:

    switch to remote branch
    1
2
    git fetch origin
git switch -c feature/login origin/feature/login

    Show local branches:

    show local branches
    1
    git branch

    Show local and remote branches:

    show all branches
    1
    git branch -a

    Delete a local branch:

    delete local branch
    1
    git branch -d feature/login

    Force delete a local branch:

    force delete local branch
    1
    git branch -D feature/login

    Delete a remote branch:

    delete remote branch
    1
    git push origin --delete feature/login

    Push a local branch that does not have a remote branch yet:

    push new branch to remote
    1
    git push -u origin feature/login

    Rename the current branch:

    rename current branch
    1
    git branch -m new-branch-name

    Show the current branch:

    show current branch
    1
    git branch --show-current

    Show merged branches:

    show merged branches
    1
    git branch --merged

    Warning

    git branch --merged only shows branches that still exist locally. If a merged source branch was already deleted, it will not appear in the output. The merged commits are still part of the history, but the branch name is gone.

    Discard file changes

    Use these commands when you want to throw away local uncommitted changes.

    Warning

    These commands discard changes from your working tree. Make sure you do not need the changes before running them.

    Discard changes in all tracked files:

    discard all tracked file changes
    1
    git restore .

    Discard changes in one file:

    discard one file
    1
    git restore path/to/file

    Discard staged changes but keep the file content:

    unstage file changes
    1
    git restore --staged path/to/file

    Discard staged and unstaged changes in one file:

    discard staged and unstaged file changes
    1
2
    git restore --staged path/to/file
git restore path/to/file

    Remove untracked files and folders:

    remove untracked files and folders
    1
    git clean -fd

    Merge

    git merge combines changes from another branch into the branch you are currently on. For example, if you are on main and merge feature/login, Git brings the commits from feature/login into main.

    Merge a branch into the current branch:

    merge branch
    1
2
    git switch main
git merge feature/login

    Merge process:

    1. Git checks the current branch.
    2. Git finds the common commit between the current branch and the branch being merged.
    3. Git applies the commits from the source branch into the current branch.
    4. If the changes do not conflict, Git completes the merge.
    5. If both branches changed the same lines, Git stops and asks you to resolve conflicts.

    Note

    If the current branch has no new commits, Git can do a fast-forward merge. If both branches have new commits, Git creates a merge commit.

    Merge without creating a merge commit only if fast-forward is possible:

    fast-forward only merge
    1
    git merge --ff-only feature/login

    Always create a merge commit:

    merge with merge commit
    1
    git merge --no-ff feature/login

    Abort a merge before finishing it:

    abort merge
    1
    git merge --abort

    Resolve merge conflicts:

    finish merge after conflicts
    1
2
3
    git status
git add path/to/resolved-file
git commit

    Rebase

    git rebase moves your branch commits so they start from a newer base commit. It is commonly used to update a feature branch with the latest changes from main while keeping a linear history.

    Rebase the current branch on top of main:

    rebase current branch
    1
2
3
    git switch feature/login
git fetch origin
git rebase origin/main

    Rebase one branch onto another branch:

    rebase branch onto main
    1
2
    git switch feature/login
git rebase main

    Rebase process:

    1. Git finds the commits that exist only on your current branch.
    2. Git temporarily removes those commits.
    3. Git moves your branch to the new base branch.
    4. Git reapplies your commits one by one.
    5. If conflicts happen, Git stops and asks you to resolve them.

    Continue after resolving rebase conflicts:

    continue rebase
    1
2
3
    git status
git add path/to/resolved-file
git rebase --continue

    Abort a rebase and return to the previous state:

    abort rebase
    1
    git rebase --abort

    Skip the current commit during a rebase:

    skip rebase commit
    1
    git rebase --skip

    Interactive rebase for editing, squashing, or reordering commits:

    interactive rebase
    1
    git rebase -i HEAD~3

    Pull remote changes using rebase instead of merge:

    pull with rebase
    1
    git pull --rebase

    Warning

    Rebase rewrites commit history. Avoid rebasing commits that were already pushed to a shared branch unless your team expects that workflow. If you rebase a pushed branch, you usually need git push --force-with-lease.
